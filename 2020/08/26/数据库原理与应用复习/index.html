数据库原理与应用复习
发表于2020-08-26|更新于2020-08-26|数据库
|字数总计:2.4k|阅读时长:8分钟

数据库系统概述

DBS：数据库系统
DB：数据库
DBMS：数据库管理系统（软件）
DBA：数据库管理员
MIS：应用程序

概念模型：人脑对客观事物的抽象
逻辑模型：数据库设计者对概念模型的转换
物理模型：DBMS对逻辑模型的转换

外模式：外部用户看数据的试图
模式：数据库设计者看数据的试图
内模式：系统程序员看数据的试图

外模式==>模式：逻辑独立性
模式==>内模式：物理独立性

概念模型

概念模型（实体-联系模型）

实体：可区分的事物
实体集：同类实体的集合
联系：实体集之间的关联

属性：实体（或联系）所具有的特征 教师实体
值域：属性的取值范围 （男，女）

型：实体或联系的结构描述 教师(工号, 姓名, 性别)
值：一组属性值的组合 (15021, ”李启沐”, 男)

候选码：区分不同实体或联系的属性
主码：可从候选码中选择

关系模型与关系运算

关系：二维表
属性：列
元组：行
关键字：列名
候选关键字：能区分行或列的组合
主键：候选关键字中的一个
外键：两张表具有相同列名，则互为外键

关系模型的数学基础

域：一组具有相同数据类型的值的集合。
笛卡尔积：ABC ACB BAC BCA CAB CBA
关系：笛卡尔积的子集，即二维表
元组：关系中的行

选择：筛选行
投影：筛选列
连接：从两张表中筛选满足一定条件的元组

等值连接：选取两张表列相等的元组
自然连接：列去重

数据完整性与关系规范化

数据完整性

  • 实体完整性：非空、不重复
  • 参照完整性：从表中有的行，主表中都有
  • 用户定义完整性：指定列的取值范围和数据类型，制定数据检查规则

函数依赖：不同X决定不同Y（学号==>姓名）
完全函数依赖：A与B列决定C列，但单独A或B不能决定C列
部分函数依赖：单独A或B即可决定C列
传递函数依赖：A列决定B列（B列不能决定A列），B列决定C列，则A列决定C列

关系规范化

  • 第一范式（1NF）：原子性
    当关系模式R的所有属性都不能分解为更基本的数据单位时。

  • 第二范式（2NF）：完全函数依赖
    如果关系模式R满足第1范式，并且R的所有非主属性都完全依赖于R的每一个候选关键属性。

  • 第三范式（3NF）：无传递函数依赖
    设R是一个满足2NF条件的关系模式，X是R的任意属性集，如果X非传递依赖于R的任意一个候选关键字。

关系模式分解既要符合范式，还要遵守以下两个原则：
无损连接：意思是分解后的多个关系，经过连接运算，能够还原为分解前的关系。
依赖保持：分解前的关系中的函数依赖，在分解后仍然得以保留。

数据库的文件集与创建数据库

用户视图：表、视图、索引、存储过程等数据库对象
物理视图：数据库文件
创建：create
修改：alter
删除：drop
分离：detach
附加：for attach

数据表的结构设计与约束设计

数据表的结构设计

  • DDL（数据定义语言）：对数据库的创建(create)、修改(alter)、删除(drop)

  • DML（数据操纵语言）：对数据的增(insert)、删(delete)、改(update)、查(select)

  • DCL（数据控制语言）：安全管理

约束设计

创建约束：[CONSTRAINT 约束名]

  • 主键约束：PRIMARY KEY
  • 外键约束：FOREIGN KEY
  • 唯一约束：UNIQUE
  • 检查约束：CHECK
  • 默认约束：DEFAULT
  • 非空约束：NOT NULL

聚合函数

count(), min(), max(), sum(), avg()等

WHERE子句

常见的关系表达式：>, <, =, between…and, in, like, is
常见的逻辑表达式：and, or, not

模糊匹配

like 和 通配符

%表示任意长度的任意字符
_表示一个字符

[^]表示不选其中字符

INTO子句

将选择的结果放在一个新表中存储起来

Group BY子句

Group BY子句用于分组统计，用Having表示筛选条件

Order by子句

用于排序，asc表示升序，desc表示降序，默认是asc

连接查询与嵌套查询

连接查询

WHERE子句 或 JOIN ON子句

  • 内连接（INNER JOIN）：等值连接、自然连接、不等值连接（INNER可省略）
  • 外连接（OUTER JOIN）：左外连接、右外连接、全外连接
  • 交叉连接（Cross JOIN）

嵌套查询

子查询在上一级查询处理之前处理
ALL：满足子查询中所有值的记录
ANY/SOME：满足集合中任意一值
IN：字段内容是否是结果集合或者子查询中的内容
EXISTS：存在返回真，否则假

视图与索引

  • 视图：类似于将SQL语句封装成一个函数，多用于查询

  • 索引：创建索引表来加速数据检索和连接、优化查询、强制实行唯一性等

索引包括：唯一性索引、主键索引、聚集索引

存储过程

存储过程：类似于将SQL语句封装成一个可传参的函数，多用于操作

事务

事务概念：事务是一个逻辑工作单元。单元中的所有操作要么全部成功，要么全部取消。事务一旦被执行，中间状态不能被其他事务获取。

  • 原子性(Atomicity)：事务中的操作要么全成功，要么全回退。
  • 一致性(Consistency)：事务执行前后数据在逻辑上是一致的。
  • 隔离性(Isolation)：A事务执行时，B事务不能得到A事务中间的状态。即事务串行化，B事务得到的数据要么是A事务执行前的状态，要么是执行后的状态。
  • 持久性(Durability)：事务执行完后，对数据进行的更改将被保存。

触发器

  1. 触发器用于保护数据一致性，就其本质而言是一种特殊的存储过程，执行插入、删除、更新操作时，DML触发器将自动触发。
  2. 按其触发其执行时间分类，有After 和 Instead of两种触发器。
  3. 在插入、删除、更新操作没有保存到数据库之前，数据表的两个临时表INSERTED和DELETED会存放临时数据，触发器可以利用这两个临时表中的数据。

SQL Server的安全机制

  • 身份验证：连接权，确认账号是否有效、口令是否正确。
  • 访问许可：访问权，验证用户是否是数据库的合法用户。
  • 操作许可：操作权，指定用户有哪些操作权限。
